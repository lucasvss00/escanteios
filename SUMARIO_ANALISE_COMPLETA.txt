RESPOSTA À PERGUNTA 3: KELLY CRITERION
═══════════════════════════════════════════════════════════════════════════════

FÓRMULA:
  Kelly = (p × b - q) / b

  onde:
    p = acurácia do modelo (probabilidade de acerto)
    b = odds - 1 (lucro líquido em caso de vitória) = 0.90
    q = 1 - p (probabilidade de erro)

CÁLCULOS:

Min 15 (Acurácia 60.4%):
  Kelly = (0.604 × 0.90 - 0.396) / 0.90 = 0.0454 = 4.54% (full Kelly)
  Half-Kelly = 2.27%

Min 30 (Acurácia 62.7%):  ← MÁXIMO
  Kelly = (0.627 × 0.90 - 0.373) / 0.90 = 0.0566 = 5.66% (full Kelly)
  Half-Kelly = 2.83%

Min 45 (Acurácia 62.6%):
  Kelly = (0.626 × 0.90 - 0.374) / 0.90 = 0.0564 = 5.64% (full Kelly)
  Half-Kelly = 2.82%

Min 60 (Acurácia 64.1%):
  Kelly = (0.641 × 0.90 - 0.359) / 0.90 = 0.0698 = 6.98% (full Kelly)
  Half-Kelly = 3.49%

Min 75 (Acurácia 61.9%):
  Kelly = (0.619 × 0.90 - 0.381) / 0.90 = 0.0285 = 2.85% (full Kelly)
  Half-Kelly = 1.43%

RECOMENDAÇÃO:

Usar HALF-KELLY (50% de full Kelly), não full Kelly:
  • Full Kelly é agressivo demais → risco de ruína da banca
  • Half-Kelly balanceia segurança com upside
  • Mais robusto a mudanças de padrão

🚨 RED FLAG #1: CONCENTRAÇÃO EXTREMA DE VOLUME

Problema:
  Min 30 + Min 60 = 31.888 apostas / 35.100 total = 90.8% DO VOLUME
  Dependência crítica de apenas 2 snapshots

Riscos Específicos:

  a) Single Point of Failure
     Se um minuto falha em produção, 45%+ do lucro desaparece

  b) Market Impact
     Alta demanda em Min 30/60 pode degradar odds

     Exemplo:
     • Esperado: odds 1.90x
     • Real com alta procura: odds 1.85x (-2.6% ROI)
     • Min 30 ROI: 14.8% → 12.2% (perda de 2.6pp)
     • Min 60 ROI: 17.4% → 14.8% (perda de 2.6pp)

     Lucro esperado cai de +15.42% para ~+12.8% (-19% impacto)

  c) Viés de Seleção
     Estes minutos têm características ESTRUTURALMENTE DIFERENTES
     • Min 30 é início de jogo (times ainda exploram)
     • Min 60 é 2º tempo (padrões mudaram)
     • Se um período muda, ambos podem degradar

Mitigação:
  ✅ Não alocar >60% em Min 30+60 combinados (está em 80%, REDUZIR)
  ✅ Diversificar com Min 45 (hedge de estrutura de intervalo)
  ✅ Monitorar correlação de erros: são realmente independentes?
  ✅ Algoritmo deve rejeitar apostas se odds < 1.88x
  ✅ Testar sensibilidade: ROI com odds 1.80, 1.85, 1.90, 1.95
  ✅ Registrar TODAS as apostas com odds reais para análise pós

⚠️ RED FLAG #3: DADOS ESCASSOS EM ALGUNS MINUTOS

Problema:
  Min 15:  2.132 apostas (muito baixo)
  Min 45:  3.660 apostas (muito baixo)
  Min 75:  4.630 apostas (baixo)
  Min 30: 16.804 apostas (OK)
  Min 60: 15.084 apostas (OK)

Impacto:
  Com dados pequenos, estatísticas não são confiáveis
  • Intervalos de Confiança (IC 90%) ficam muito largos
  • Acurácia pode estar inflada (overfitting)
  • Sharpe-like ratio não é robusto

  Exemplo (Min 15):
    IC 90% = [7.2%, 13.6%] → ±3.2pp de variância
    Isso significa que ROI real pode ser 7.2% ou 13.6% (ainda incerto)

Evidência de Dados Pequenos:
  Min 15: amplitude de 41.1% (extrema)
  Min 45: amplitude de 29.3% (muito alta)
  Min 75: amplitude de 16.1% (alta)

  Vs. Min 30: amplitude de 4.1% (pequena - dados grandes)

Mitigação:
  ✅ Coletar +50% mais dados antes de escalar
     - Min 15: de 2.132 para 3.200+ apostas
     - Min 45: de 3.660 para 5.500+ apostas
     - Min 75: de 4.630 para 7.000+ apostas

  ✅ Aumentar k-folds de 5 para 10 (ou 15)
     - Melhor estimativa de variância
     - Menos dependência de um fold único

  ✅ Usar TimeSeriesSplit (não stratified k-fold)
     - Respeita ordem temporal dos dados
     - Mais realista que fold aleatório

  ✅ Validação em período holdout
     - Separar últimas 2 semanas como teste
     - Nunca visto antes pelo modelo
     - True out-of-sample

⚠️ RED FLAG #5: PADRÕES SAZONAIS NÃO ANALISADOS

  Modelo ASSUME que padrões de escanteios são estacionários (não mudam)
  Realidade: Padrões VARIAM por liga, competição, temporada, etc.

  3. Por Características do Campo

  4. Mudanças Sazonais (Injury, Roster, Táticas)
     • Lesão de jogador-chave pode mudar dinâmica
     • Novas contratações mudam estilo da equipe

Risco:
  • Model degradation ao longo do tempo
  • Performance pode cair 30-50% em competição diferente
  • Dataset de treino pode não representar futuro

Mitigação:
  ✅ Estratificar backtesting por liga/competição
     - Treinar separado para cada liga?
     - Testar: Min 30 em Champions vs Brasileiro

  ✅ Testar degradação com dados antigos
     - Treinar com dados de 3 meses atrás
     - Testar em dados de hoje
     - Ver quanto performance degrada

  ✅ Reestimar modelo a cada 2-4 semanas
     - Incorporar novos dados, esquecer padrões antigos
     - Detectar mudanças estruturais cedo

  ✅ Separar dataset por competição principal
     - Submodelo para Champions?
     - Submodelo para Liga Nacional?

  ✅ Análise de Feature Importance
